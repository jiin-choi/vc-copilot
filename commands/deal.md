# Deal Finder — Alpha Sourcing

남들이 안 보는 성장 궤적을 먼저 감지합니다. Pre-Consensus → Early Growth → Consensus 3-Tier 시그널 피라미드로 **VC Blind Spot 기업**을 우선 포착합니다.
**설계 원칙**: 부재(不在)가 시그널 — 성장하는데 VC 관심 없는 기업 = 최고 알파.

## 심사역 선호 프로파일
MEMORY.md 선호 프로파일 + CLAUDE.md 섹터·스테이지 기준 적용.

---

## 0단계: 워치리스트 로드

MEMORY.md `## Deal Watchlist`를 읽어 기존 추적 기업 목록을 로드합니다.
- Active/Monitoring 상태 기업명을 Tier 2-3 쿼리의 키워드로 포함 (추가 쿼리 없이)
- Stale 기업(4주+ 시그널 없음)은 자동 제거 대상으로 표시

---

## 1단계: 3-Tier 시그널 스캔 (병렬 WebSearch, 8 쿼리)

3-Tier 피라미드를 **직접 병렬 WebSearch**로 수행합니다 (Task 에이전트 사용하지 않음). MEMORY.md 섹터 선호도에서 섹터 키워드를 로드합니다.

### Tier 1 — Pre-Consensus (최고 알파, 3 쿼리)
남들이 안 보는 기업을 찾는 쿼리. VC 펀딩 뉴스에 나오지 않는 성장 기업.

| # | 쿼리 | 찾는 것 |
|---|------|---------|
| 1 | `{섹터} 스타트업 흑자 OR 매출성장 OR BEP -투자유치 -펀딩 {년도}` | 수익성 입증 but VC 미관여 |
| 2 | `스타트업 부트스트래핑 OR 자체성장 OR 무차입 {년도} 한국` | 자기자본 성장 기업 |
| 3 | `{섹터} 매출 100억 OR 흑자전환 -시리즈 site:platum.kr OR site:thevc.kr {년도}` | 재무 마일스톤 but 펀딩 뉴스 X |

- 수집: 기업명, 매출/수익성 지표, 섹터, 성장 시그널 유형
- `{섹터}`에는 MEMORY.md 섹터 선호도에서 관심도 '높음' 이상 섹터를 순환 사용

### Tier 2 — Early Growth (중간 알파, 3 쿼리)
펀딩 발표 3~6개월 전의 선행 지표.

| # | 쿼리 | 찾는 것 |
|---|------|---------|
| 4 | `스타트업 CTO VP 영입 {년도} {월}` + `사옥 이전 OR 사무실 확장` | 스케일업 준비 시그널 |
| 5 | `스타트업 대기업 제휴 MOU 파트너십 {년도} {섹터}` | B2B 검증 시그널 |
| 6 | `스타트업 해외진출 OR 글로벌 OR 수출 {년도} 한국 {섹터}` | 해외 확장 = 대형 라운드 선행 |

- 수집: 기업명, 시그널 유형(채용/제휴/해외진출), 관련 섹터
- 워치리스트 Active/Monitoring 기업명을 쿼리에 OR로 추가 (추가 쿼리 없이 교차 확인)

### Tier 3 — Consensus (확인용, 2 쿼리)
기존 펀딩 쿼리 유지. **발견용이 아닌 교차 검증용**으로만 사용.

| # | 쿼리 | 역할 |
|---|------|------|
| 7 | `스타트업 시리즈A 시리즈B 투자 {년도} {월} site:platum.kr OR site:thevc.kr` | 이미 알려진 딜 확인 |
| 8 | `앱스토어 순위 급상승 OR MAU 급증 스타트업 {년도} {월}` | 성장 확인 |

- 수집: 기업명, 라운드, 금액, 투자자, 성장 지표

## 2단계: Negative Screen + Alpha Score 랭킹

### 2-1. 시그널 통합 + Tier 태깅
- 3개 Tier 수집 기업을 통합하고 각 기업에 포착된 Tier 태깅
- 복수 Tier에서 포착된 기업은 **수렴(Convergence)** 표시

### 2-2. Negative Screen — 컨센서스 레벨 태깅
Tier 1-2 포착 기업에 대해 **"이미 VC 관심을 받고 있는가?"** 체크:
- Tier 3 결과에 동일 기업 등장 여부 확인
- 검색 결과 내 주요 VC명 언급 빈도 체크
- Deal DB 중복 체크

| 레이블 | 정의 |
|--------|------|
| **VC Blind Spot** | Tier 1 + VC 언급 0 + Deal DB 미등록 |
| **Under-Watched** | Tier 1-2 + VC 언급 1~2 |
| **Early Consensus** | Tier 2-3 + VC 언급 일부 |
| **Full Consensus** | Tier 3 + VC 언급 다수 |

### 2-3. Alpha Score 산출
```
Alpha Score = Tier 가중치 + 수렴 보너스 + 선호도 매칭 - Drop 패턴 - 컨센서스 감점

Tier 가중치:    T1=3, T2=2, T3=1  (최고 Tier 기준)
수렴 보너스:    T1+T2=+3, T1+T3=+2, T2+T3=+1, 전 Tier=+4
선호도 매칭:    강한 선호 시그널 +2, 중간 +1, 비선호 -2  (MEMORY.md 기준)
Drop 패턴:     매칭 시 -1/건  (제외하지 않음, 경고만)
컨센서스 감점:  Full=-2, Early=-1, Under=0, Blind Spot=+1
```

### 2-4. 분류
| 분류 | Alpha Score | 설명 |
|------|-------------|------|
| **Alpha Pick** | 8+ | 최우선 리서치 대상 |
| **Emerging** | 5-7 | 유망, 추가 시그널 대기 |
| **Consensus** | 3-4 | 이미 알려진 딜, 참고용 |
| **기타** | <3 | 약한 시그널 |

### 2-5. 컨트래리언 픽 (1개)
선호 프로파일에 **반하지만** Alpha Score가 높은 기업 1개를 강제 포함. 편향 자기강화를 차단하는 디바이어싱 장치.
- 비선호 시그널에 해당하지만 복수 Tier 포착 또는 정량 지표가 눈에 띄는 기업
- "왜 반대 포지션인지"와 "그럼에도 주목할 근거"를 명시

### 2-6. 포트폴리오사 체크
기존 포트폴리오사는 별도 표시

## 3단계: 기업 카드 작성 (병렬, 간결)

**Alpha Pick + Emerging 상위 3~5개 + 컨트래리언 픽 1개** 기업에 대해 간결한 리서치. 기업당 WebSearch 1~2회로 제한합니다.
- **상위 3개 기업**: **병렬 Task 에이전트** (최대 3개)로 리서치 + Slack 검색
- **나머지 기업**: 메인 컨텍스트에서 직접 WebSearch로 처리
- Consensus/기타 기업은 스캔 단계에서 수집된 정보만으로 간략 카드를 작성합니다.

### 각 기업별:
- WebSearch: `"{기업명}" 투자 펀딩 매출 서비스` — 핵심 정보 한번에
- Slack 검색으로 기업명 검색 — 내부 논의 이력 확인
- DART 비상장 감사보고서에서 매출/영업이익 확인 시도 (가능한 경우만)

### 기업 카드 포맷
```
#### {기업명} — {한줄 설명}
- **컨센서스 레벨**: {VC Blind Spot / Under-Watched / Early Consensus / Full Consensus}
- **Alpha Score**: {점수} ({산출 근거: T1=3, 수렴=+3, 선호=+2, ...})
- **포착 시그널**: {어떤 Tier, 어떤 쿼리에서 포착}
- **사업**: {무엇을 하는 기업}
- **핵심 지표**: {매출, 사용자, 성장률 등}
- **펀딩**: {최근 라운드, 금액, 투자자}
- **DART 재무**: {매출/영업이익 — 확인 가능 시에만}
- **왜 지금**: {이 기업을 지금 봐야 하는 이유}
- **리스크**: {핵심 리스크 1~2줄}
- **Drop 패턴 매칭**: {MEMORY.md Drop 패턴과 매칭되는 항목 — 없으면 생략}
- **Slack 인텔**: {내부 이력 — 없으면 "없음"}
```

## 4단계: 터미널 출력 → 기업 선택

아래 포맷으로 **터미널에 직접 출력**합니다 (Notion 저장 없음):

```
## Deal Finder — {날짜}
> Pre-Consensus → Early Growth → Consensus

스캔: T1 {N}건 / T2 {N}건 / T3 {N}건 → 총 {N}개 포착

### Watchlist Alerts
> 새 시그널 감지 시에만 표시. 없으면 이 섹션 생략.

| 기업명 | 이전 시그널 | 새 시그널 | 누적 횟수 | 액션 제안 |

### Alpha Picks (Score 8+)
{기업 카드 — VC Blind Spot / Under-Watched 태그 포함}

### Emerging (Score 5-7)
{간결한 카드}

### Consensus (Score 3-4) — 참고용
| 기업명 | 섹터 | 시그널 | 펀딩 정보 | Score | 비고 |

### 컨트래리언 픽
{선호 프로파일에 반하지만 시그널이 강한 기업 1개 — 상세 카드 + "왜 반대 포지션인지" + "그럼에도 주목할 근거"}

### 기존 파이프라인
| 기업명 | Status | 이번 시그널 |

### Watchlist 변동
| 기업명 | 변동 | 사유 |
```

출력 후 **AskUserQuestion** `multiSelect: true`로 기업 선택:
- **모든 분류 그룹 기업** 선택지에 포함 (질문당 2~4개, 기업 수에 맞춰 분할)
- 필터링 제외 기업도 심사역이 직접 선택할 수 있도록 전체 제시

## 5단계: 선택된 기업 → Deal DB + 콜드메일

선택된 기업에 대해:
- **Notion Deal DB** 등록 (기존 기업은 토글 추가)
  - `Source`: `Deal Finder — {날짜}`
  - 토글 제목: `[{작성일} Deal Finder] {기업명} 리서치`
- **Gmail 초안 생성**: 콜드메일 템플릿 + 기업 카드의 "사업"+"왜 지금" 기반 맞춤 인사이트 1~2문장

## 6단계: 선호도 학습 + 워치리스트 업데이트

### 6-1. 선호도 학습
MEMORY.md 크로스 스킬 학습 + 보수적 학습 룰 적용. 동일 패턴 2회+ 확인 시에만 프로파일 반영.

### 6-2. 워치리스트 업데이트 (MEMORY.md `## Deal Watchlist`)
- **Auto-add**: Alpha Score 5+ & Tier 1 포착 기업 → Active 상태로 추가
- **Manual add**: 사용자가 선택한 기업 중 Deal DB 미등록 → Active 상태로 추가
- **Graduate**: Deal DB에 추가된 기업 → 워치리스트에서 제거
- **Re-signal**: 기존 워치리스트 기업에 새 시그널 → 최근 시그널/횟수 업데이트, Active로 복귀
- **Aging 규칙**:
  - Active: 2주 이내 시그널
  - Monitoring: 2~4주 시그널 없음
  - Stale: 4주+ 시그널 없음 → 자동 제거
- **최대 15개** 유지. 초과 시 Stale → Monitoring 순으로 제거
