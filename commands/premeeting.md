# 미팅 사전 자료 준비

오늘 예정된 투자 미팅에 대한 사전 자료를 자동으로 준비합니다.

## 실행 순서

### 1단계: 미팅 일정 가져오기 (오늘만)
- Google Calendar MCP (`list-events`)로 **오늘** 일정을 조회합니다.
  - `calendarId`: `primary`
  - `timeZone`: `Asia/Seoul`
  - 오늘 날짜 00:00 ~ 오늘 날짜 23:59 (1일간)
- 일정 제목에 `[미팅]` 또는 `[IR]`이 포함된 일정만 필터링합니다.
- 각 일정에서 **기업명** (태그 뒤의 텍스트), **시간**, **장소**를 추출합니다.
  - 예: `[미팅] 스타트업A` → 기업명: `스타트업A`
  - 예: `[IR/소개자] 스타트업B` → 기업명: `스타트업B`

### 2단계: 미팅이 없는 경우
- `[미팅]` 또는 `[IR]` 일정이 없으면 "오늘 예정된 미팅이 없습니다."라고 안내하고 종료합니다.

### 3단계: 기업별 사전 자료 생성
각 기업에 대해 아래를 **병렬로** 수행합니다 (**최대 3개 Task 에이전트**. 4개 이상 기업 시 3개씩 배치 처리):

#### 3-1. Notion Deal DB 확인
- Notion API로 기업명을 검색합니다.
- 기업 페이지가 있으면:
  - **Status**를 **`Meeting`으로 자동 변경**합니다.
  - **IR Deck** 첨부파일이 있는지 확인합니다.
  - 기존 페이지의 블록 내용을 읽어 이미 작성된 정보를 파악합니다.
- 기업 페이지가 없으면:
  - 나중에 4단계에서 새 페이지를 생성합니다.

#### 3-2. Slack 검색
Slack 검색으로 기업명 검색 → 팀 내부 검토 이력 수집.
- **수집 정보**: 검토 타임라인, 담당 심사역, 검토 조건(밸류·금액), 투심 결과, Drop/Hold 사유 등
- 검색 결과가 있으면 사전 자료에 "Slack 인텔" 섹션으로 포함합니다.
- 특히 **이전 Drop/Hold 사유**, **밸류에이션 논의**, **팀 내부 의견**은 미팅 체크포인트에 반영합니다.

#### 3-3. 웹 리서치
WebSearch 도구로 다음을 조사합니다:
- `"{기업명}" 스타트업` — 기업 개요, 서비스 설명
- `"{기업명}" 투자 펀딩` — 펀딩 이력, 밸류에이션
- `"{기업명}" site:thevc.kr OR site:platum.kr OR site:besuccess.com` — 스타트업 전문 매체 기사
- MEMORY.md Watchlist에 해당 기업이 있으면 포착 이력을 사전 자료에 포함

#### 3-4. IR Deck 분석 (있는 경우)
- Notion에 IR Deck이 첨부되어 있으면 내용을 분석합니다.
- PDF인 경우 Python PyPDF2로 텍스트를 추출하여 분석합니다.

### 4단계: 사전 자료 작성
수집한 정보를 아래 포맷으로 **간결하게** 정리합니다.

**분량 규칙**: 기업당 터미널 출력 **30줄 이내**. 미팅 전 2분 안에 훑을 수 있는 분량.
- 각 섹션은 bullet 3~5개 이내. 장황한 설명 금지.
- 대표 이력은 한 줄 요약. 별도 섹션 만들지 않음.
- 제휴/파트너십, 호스트 라인업 등은 기업 개요에 핵심만 병합.
- 시장 분석은 TAM + 벤치마크 1줄 + 경쟁 1줄.
- 체크포인트는 **5개 이내**, 미팅에서 반드시 물어볼 것만.

```
## {기업명} 미팅 사전 자료
{날짜} {시간} | {장소}

### 기업 개요
- 설립/대표: {연도}, {이름} ({핵심 이력 한 줄})
- 서비스: {한 줄 설명}
- 주요 지표: {매출, 트랙션 등 핵심 수치}
- 펀딩: {현재 라운드 + 밸류 + 주요 투자자}

### Slack 인텔
- {소스/담당/내부 온도 — 핵심만 2~3줄}

### 체크포인트
- [ ] {미팅에서 꼭 확인할 질문 5개 이내}

### 적합도
- 섹터/스테이지 한 줄 판정 + 핵심 리스크/기회 각 1줄
```

### 5단계: Notion 저장
- 토글 제목: `[{날짜} {미팅/IR} 사전자료] {시간} [{태그}] {기업명} | {장소}`
- Deal DB 중복 체크: 기존 페이지 → 토글 추가, 없으면 → 새 페이지 생성

### 6단계: 터미널 전문 출력
Notion 저장 내용과 동일한 전문 마크다운 출력 + Notion 페이지 링크.

## 주의사항
- 미확인 정보 → "확인 필요" 표시 / 출처 필수 / 결론 단정 금지
- 반론·리스크 반드시 포함
