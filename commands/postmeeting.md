# 투자 메모 생성 (Post-Meeting)

미팅 후 투자 메모를 생성합니다. 녹취록, IR Deck을 종합하여 구조화된 투자 메모를 작성합니다.

## 실행 순서

### 1단계: 대상 미팅 선택
- Google Calendar MCP (`list-events`)로 **최근 7일간** 일정을 조회합니다.
  - `calendarId`: `primary`
  - `timeZone`: `Asia/Seoul`
  - 오늘 기준 D-7 ~ D 범위
- 일정 제목에 `[미팅]` 또는 `[IR]`이 포함된 일정만 필터링합니다.
- **AskUserQuestion**으로 대상 기업을 선택받습니다:
  - 최근 7일 미팅 목록을 옵션으로 제시
  - 복수 선택 가능 (`multiSelect: true`)
  - "Other" 옵션으로 직접 기업명 입력도 가능

### 2단계: 미팅이 없는 경우
- 최근 7일간 `[미팅]` 또는 `[IR]` 일정이 없으면 "최근 7일간 미팅 일정이 없습니다."라고 안내하고 종료합니다.

### 3단계: 입력 소스 수집
선택된 기업별로 아래를 수행합니다:

#### 3-1. Notion 페이지 확인
- Notion API로 기업 페이지를 찾습니다.
- 해당 페이지의 **블록 내용**을 읽어옵니다 (기존 premeeting 자료, 녹취록 등).
- **IR Deck** 첨부 여부를 확인합니다.
- IR Deck이 있으면 다운로드 후 PDF 텍스트 추출합니다.

#### 3-2. 녹취록 확인
- Notion 페이지 블록에서 녹취록/미팅 노트를 탐색합니다.
- **Notion AI 녹음(transcription) 블록 제약**: `type=unsupported`로 반환되며 API로 읽기 불가. 이 블록이 감지되면:
  - 심사역에게 안내: "Notion AI 녹음 트랜스크립트가 있지만 API로 읽을 수 없습니다. Notion에서 트랜스크립트를 일반 텍스트 블록으로 복사해주시면 읽을 수 있습니다."
- 녹취록(일반 텍스트)도 없고 AI 녹음만 있는 경우, **AskUserQuestion**으로 심사역에게 확인합니다:
  - "녹취록이 Notion AI 녹음에만 있어 API로 읽을 수 없습니다. 어떻게 진행할까요?"
  - 옵션: "Notion에서 트랜스크립트를 텍스트로 복사 후 재실행", "미팅 메모를 터미널에 직접 입력", "녹취 없이 진행"

#### 3-3. Slack 검색
Slack 검색으로 기업명 검색 → 팀 내부 논의 이력 수집.

#### 3-4. 웹 리서치 (보충)
- premeeting에서 수집하지 못한 최신 정보를 WebSearch로 보충합니다.
- 특히 최근 뉴스, 경쟁사 동향, 시장 데이터를 업데이트합니다.

### 4단계: 투자 메모 작성
수집한 모든 소스를 종합하여 아래 구조로 투자 메모를 작성합니다.
premeeting에서 이미 다룬 사업 분석·시장·팀은 반복하지 않고, **미팅에서 새로 확인된 내용 + 판단 재료**에 집중합니다.

```
## {기업명} 투자 메모
작성일: {날짜} | 미팅일: {미팅 날짜} {시간}

### 1. 미팅 핵심 (새로 알게 된 것)
- {창업자가 강조한 포인트}
- {기존 자료와 다른 점 / 업데이트된 내용}
- {논리적 허점 — 주장 vs 데이터 괴리, 답변 회피 영역}

### 2. 정량 체크
> IR Deck / 녹취에서 확인 가능한 수치 기반. 미확인 항목은 "확인 필요" 표시.

| 지표 | 수치 | 판단 |
|------|------|------|
| 매출 / 성장률 (MoM·YoY) | | |
| 리텐션 (코호트·DAU/MAU) | | |
| 유닛이코노믹스 (CAC·LTV·Payback) | | |
| 마켓 효율성 (Burn Multiple·Rule of 40) | | |

### 3. 투자 판단 재료
| Bull Case | Bear Case |
|-----------|-----------|
| {포인트 1} | {리스크 1} |
| {포인트 2} | {리스크 2} |
| {포인트 3} | {리스크 3} |

**스케일업 시나리오**: {1조 도달 경로 + 필요 조건 + 현실성 판단}

**킬 시나리오**
1. {리스크 1 + 근거}
2. {리스크 2 + 근거}
3. {리스크 3 + 근거}

### 4. 밸류에이션
- 현재 라운드: {시리즈, Pre/Post, 금액}
- 비교 기업: {유사 기업 밸류에이션}
- 적정성 판단 재료: {멀티플, 성장률 대비 평가}

### 5. 후속 액션
- [ ] {액션 1}
- [ ] {액션 2}
- [ ] {액션 3}
```

### 5단계: Notion 저장
- 토글 제목: `[{날짜} 투자메모] {기업명}`
- Deal DB 중복 체크: 기존 페이지 → 토글 추가, 없으면 → 새 페이지 생성
- **Status는 변경하지 않습니다**. 기존 상태를 그대로 유지합니다.

### 6단계: 터미널 전문 출력
Notion 저장 내용과 동일한 전문 마크다운 출력 + Notion 페이지 링크.

## 주의사항
- 미확인 정보 → "확인 필요" 표시 / 출처 필수 / 결론 단정 금지
- 반론·리스크 반드시 포함
- 정량 분석에서 데이터가 부족한 경우, 어떤 데이터가 필요한지 명시하고 후속 액션에 포함합니다.
