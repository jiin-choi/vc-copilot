# IPO Tracker — 예비심사 기업 위클리 트래커

KRX KIND 예비심사 현황을 크롤링하여 Google Sheets에 업데이트합니다.
순수 스팩을 제외한 실질 기업만 트래킹하고, 신규 기업은 VC 분석 리서치를 수행합니다.

**언제 쓰나**: 매주 목요일 오후 4시 (또는 수동 실행). `/ipo`로 실행.

인자: $ARGUMENTS (없으면 위클리 업데이트 모드)

---

## 1단계: KRX KIND 크롤링

Python `urllib`로 KRX KIND 예비심사기업 목록을 조회합니다.

### 1-1. 데이터 수집
```python
url = 'https://kind.krx.co.kr/listinvstg/listinvstgcom.do'
params = {
    'method': 'searchListInvstgCorpSub',
    'forward': 'listinvstgcom_sub',
    'searchCorpName': '',
    'pageIndex': '1',
    'currentPageSize': '200',
    'orderMode': '1',
    'orderStat': 'D',
    'fromDate': '{6개월 전 YYYYMMDD}',
    'toDate': '{오늘 YYYYMMDD}',
    'marketType': '',
    'listTypeArr': '',
    'invstgRsltArr': '',
    'listTypeArrStr': '',
    'invstgRsltArrStr': ''
}
```
- POST 요청, `Content-Type: application/x-www-form-urlencoded`
- `User-Agent: Mozilla/5.0`, `Referer: https://kind.krx.co.kr/...`

### 1-2. HTML 파싱
정규식으로 `<tr>` 행 파싱:
- 기업명: `title="..."` 속성
- 시장구분: `icn_t_ko.gif` → 코스닥, `icn_t_yu.gif` → 유가증권
- 상장유형, 청구일, 결과확정일, 심사결과, 상장주선인: `<td>` 텍스트

### 1-3. 필터링
**포함 기준** (청구 + 승인):
- 심사상태가 `청구서 접수` 또는 `심사 승인`
- **청구일이 올해 1월 1일 이후** OR **승인일이 올해 1월 1일 이후** (실행 시점의 연도 기준 동적 계산)
- 전년 청구 + 아직 접수 상태(올해 이벤트 없음) → 제외. 승인 시 자동 포함됨

**제외 기준** (순수 스팩):
- 기업명에 `스팩` 포함 → 제외
- 상장유형이 `스팩 소멸합병` / `스팩 존속합병`이지만 기업명에 `스팩` 미포함 → **포함** (실질 기업의 스팩 합병 상장)

## 2단계: 기존 시트와 비교

### 시트 설정
> Google Sheets ID, GID, OAuth 토큰 경로는 CLAUDE.md 또는 환경 설정에서 관리합니다.

### 시트 구조 (A~O, 15개 컬럼)
| 컬럼 | 필드 | 출처 |
|------|------|------|
| A | 기업명 | KRX |
| B | 시장구분 | KRX |
| C | 상장유형 | KRX |
| D | 심사상태 | KRX |
| E | 청구일 | KRX |
| F | 승인일 | KRX |
| G | 상장일(예정) | 리서치 |
| H | 대표주관사 | KRX |
| I | 업종/섹터 | 리서치 |
| J | 매출(억 원, 기준연도) | 리서치. 값에 연도 병기 (예: "312 ('24)") |
| K | 영업이익(억 원, 기준연도) | 리서치. 값에 연도 병기 (예: "20 ('24)") |
| L | 주요 투자자 | 리서치 |
| M | 추정 밸류(억 원) | 리서치 |
| N | 코멘트 | 팀 수동 |
| O | 최종 업데이트 | 자동 |

### 비교 로직
1. 시트 A열(기업명) 전체 읽기
2. KRX 크롤링 결과와 대조:
   - **신규**: 시트에 없는 기업 → 행 추가 (3단계 리서치 후)
   - **상태 변경**: 시트에 있지만 심사상태가 변경된 기업 → D열(심사상태) + F열(승인일) + G열(상장일) + O열(최종 업데이트) 갱신. **청구→승인 변경 시 3단계 리서치 대상에 포함**
   - **기존 동일**: 변동 없음 → 스킵

## 3단계: 신규 기업 VC 리서치 + 상장일 조사

신규 기업 및 **상태 변경(청구→승인) 기업**을 리서치합니다.

### 3-A. 신규 기업 — 전체 항목 조사
| 항목 | 컬럼 | 설명 |
|------|------|------|
| 업종/섹터 | I | 핵심 사업 영역 한 줄 |
| 매출 | J | 최근 연도, 억 원 단위. 연도 병기 (예: "312 ('24)") |
| 영업이익 | K | 최근 연도, 억 원 단위. 연도 병기 (예: "20 ('24)") |
| 주요 투자자 | L | VC/PE 투자자 (알려진 것만) |
| 상장일(예정) | G | 공모 일정 또는 뉴스 기반 예상 상장일 (YYYY-MM-DD) |
| 추정 밸류 | M | 최근 라운드 기업가치 또는 IPO 추정 시총 (억 원) |

### 3-B. 상태 변경(청구→승인) 기업 — 상장일(예정) 조사
- 승인 직후이므로 공모 일정·뉴스로 상장 예정일 WebSearch
- 확정일 → `YYYY-MM-DD`, 추정 → `YYYY-MM-DD (추정)`, 미확인 → `확인 필요`

### 실행 방식
- 리서치 대상 1~6개: 직접 WebSearch (Task 에이전트 없이)
- 리서치 대상 7개 이상: **최대 3개 Task 에이전트**로 배치 분산
- 찾을 수 없는 정보: `"확인 필요"` 기입

## 4단계: Google Sheets 업데이트

### 4-1. 신규 기업 추가
- 행 데이터: `[기업명, 시장, 유형, 상태, 청구일, 승인일, 상장일(예정), 주관사, 섹터, 매출, 영익, 투자자, 밸류, "", "{YYYY-MM-DD}"]`
- 코멘트(N)는 빈칸 (팀이 수동 입력)

### 4-2. 기존 기업 상태 갱신
- 변경된 행의 D, F, G, O열만 업데이트
- **N열(코멘트)은 절대 덮어쓰지 않음** (팀 수동 입력 보호)

### 4-3. 정렬
- 업데이트 후 시트 데이터를 E열(청구일) 기준 **내림차순** 정렬 (최신이 위)

## 5단계: IPO 임플리케이션 시트 업데이트

이번 주 변동 사항과 전체 예비심사 현황을 종합하여 월별 임플리케이션을 생성·갱신합니다.

### 시트 구조 (A~G, 7개 컬럼)
| 컬럼 | 필드 | 설명 |
|------|------|------|
| A | 시기 | 월 단위 (예: "2026년 2월") |
| B | 변동 요약 | 해당 월 누적 변동. 주차별로 구분하여 기록 |
| C | 섹터 파이프라인 시그널 | IPO 신청 섹터 패턴 → 소싱 방향 시사점 |
| D | 밸류에이션 벤치마크 | 공모 밸류 기반, 딜 검토 시 comp 활용 데이터 |
| E | 엑시트·전략적 시사점 | 포트폴리오 엑시트 타이밍, LP 커뮤니케이션 등 실행 시사점 |
| F | 심사역 코멘트 | 팀 수동 |
| G | 최종 업데이트 | 자동 (YYYY-MM-DD) |

### 업데이트 로직

#### 5-1. 기존 월 행 확인
- A열에서 현재 월 문자열 (예: "2026년 2월") 검색
- 있으면 → **해당 행 갱신** (5-3)
- 없으면 → **신규 행 추가** (5-2)

#### 5-2. 신규 월 행 추가
- 헤더 바로 아래(2행)에 삽입 → **최신 월이 항상 위**에 오도록 유지

#### 5-3. 기존 월 행 갱신
- B열(변동 요약): 기존 텍스트에 이번 주 변동을 **줄바꿈 후 append** (기존 내용 보존)
- C~E열(임플리케이션 3축): 해당 월 전체 변동을 종합하여 **덮어쓰기** (최신 분석으로 갱신)
- **F열(심사역 코멘트)은 절대 덮어쓰지 않음** (팀 수동 입력 보호)
- G열: 오늘 날짜로 갱신

#### 5-4. 임플리케이션 생성 기준

**B열 — 변동 요약** (팩트 기반, 주차 구분):
```
[2월 4주차] 신규 청구 2건 (A사·B사) / 승인 1건 (C사)
[2월 3주차] 신규 청구 1건 (D사) / 상태 변경 없음
```

**C열 — 섹터 파이프라인 시그널** (분석):
- 이번 달 신규 청구·승인 기업의 섹터 분포 패턴
- 특정 섹터 클러스터링 여부 (예: "반도체 장비 3건 연속")
- 소싱 방향 시사점

**D열 — 밸류에이션 벤치마크** (정량):
- 신규/승인 기업의 추정 밸류·매출·영익 기반 멀티플
- 동종 비상장 기업 투자 시 comp로 활용 가능한 수치
- 섹터별 IPO 밸류에이션 레인지

**E열 — 엑시트·전략적 시사점** (액션):
- IPO 윈도우 개폐 시그널 (승인 속도, 섹터별 수용도)
- 포트폴리오 중 IPO 엑시트 가능성 있는 기업에 대한 시사점
- LP 커뮤니케이션에 활용할 수 있는 시장 데이터 포인트
- 구체적 실행 권고 1~2개

### 변동 없는 주
- 이번 주 신규·상태 변경이 **0건**이면 B열에 `[n월 n주차] 변동 없음` 추가
- C~E열은 갱신하지 않음 (기존 임플리케이션 유지)

## 6단계: 터미널 출력

### 위클리 서머리
```
## IPO Tracker — 예비심사 위클리 ({YYYY-MM-DD})

### 이번 주 변동
| 구분 | 건수 | 기업 |
|------|------|------|
| 신규 청구 | {n} | {기업1, 기업2, ...} |
| 심사 승인 | {n} | {기업1, 기업2, ...} |
| 상태 변경 | {n} | {기업1 (청구→승인), ...} |

### 신규 기업 리서치
| 기업명 | 섹터 | 매출 | 영업이익 | 투자자 | 밸류 |
|--------|------|------|---------|--------|------|
| {기업1} | {섹터} | {매출 ('YY)} | {영익 ('YY)} | {투자자} | {밸류} |

### 이달의 임플리케이션 ({n}월)
**섹터 파이프라인 시그널**: {C열 내용 요약}
**밸류에이션 벤치마크**: {D열 내용 요약}
**엑시트·전략적 시사점**: {E열 내용 요약}

### 전체 현황
- 트래킹 중: {n}건 (청구 {n} / 승인 {n})

> 출처: KRX KIND 예비심사기업 현황
```

## 주의사항
- 미확인 정보 → "확인 필요" 표시 / 출처 필수
- 팀이 수동 입력한 코멘트(N, F열)는 절대 덮어쓰지 않음
- KRX 크롤링 실패 시 에러 메시지 출력 + 재시도 안내
- 순수 스팩(기업명에 "스팩" 포함)은 항상 제외
- 숫자 데이터는 천단위 콤마 없이 숫자만 입력 (시트 서식은 시트에서 처리)
