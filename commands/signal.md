# Signal Radar — 위클리 시그널 스캔

매주 전 섹터를 넓게 스캔하여 초기 투자 시그널을 포착합니다. 펀딩, 인재 흐름, 소셜, 미디어, 규제/정책 5개 카테고리를 종합하여 위클리 브리핑을 생성합니다.
**읽기 전용**: Deal DB 추가, Gmail 초안, 기업 선택, 선호도 학습은 수행하지 않습니다.

## 실행 순서

---

## 1단계: 위클리 스캔 (5개 카테고리 직접 병렬 WebSearch)

5개 시그널 카테고리를 **직접 병렬 WebSearch**로 수행합니다 (Task 에이전트 사용하지 않음 — 검색 쿼리만이므로 직접 호출이 더 효율적).

### 1-1. 펀딩 시그널
- WebSearch: `한국 스타트업 투자 유치 이번주 {년도}`, `스타트업 펀딩 site:platum.kr OR site:thevc.kr`
- WebSearch: `스타트업 시리즈A 시리즈B 투자 {년도} {월}`, `벤처투자 주간 동향`
- 최근 1주간 주요 투자 유치 건 수집 (기업명, 라운드, 금액, 투자자)
- 특이점 탐지: 이례적 밸류, 새로운 투자자 진입, 섹터 집중 현상

### 1-2. 인재 흐름
- WebSearch: 주요 테크기업 핵심 인력(리드급+) 이동 감지
- 창업 시그널: "전 {기업} {직책}" 패턴 탐지

### 1-3. 소셜 시그널
- WebSearch: `GitHub trending Korea {년도}`, `앱스토어 순위 급상승 앱 한국`, `ProductHunt 한국 스타트업`
- WebSearch: `스타트업 대규모 채용 {년도}`, `시리즈A 채용 급증`
- WebFetch: `https://github.com/trending?since=weekly` — GitHub 위클리 트렌딩 크롤링
- 채용 급증 시그널, 앱 순위 급등, GitHub 스타 급증 탐지

### 1-4. 미디어 시그널
- WebSearch: 최근 1주 스타트업 뉴스 — `스타트업 site:platum.kr`, `스타트업 site:thevc.kr`, `스타트업 site:besuccess.com`, `스타트업 site:byline.network`
- WebSearch: `스타트업 site:openads.co.kr`, `스타트업 신규 서비스 런칭 {년도}`
- 주요 기사 요약: 기업명, 핵심 내용, 시그널 유형
- 반복 언급 기업 탐지 (복수 매체에서 동시 보도 = 강한 시그널)

### 1-5. 규제/정책 시그널
- WebSearch: `신산업 규제 완화 {년도}`, `스타트업 정책 {년도}`, `규제 샌드박스 신규`
- WebSearch: `금융규제 핀테크 {년도}`, `데이터 규제 개인정보 {년도}`, `플랫폼 규제 {년도}`
- 새로운 시장을 여는 규제 변화 포착, 영향받는 섹터 분석

---

## 2단계: 시그널 프로세싱

### 2-1. 클러스터링
- 수집된 시그널을 **기업/섹터 단위로 클러스터링**합니다.
- 동일 기업이 복수 카테고리에서 포착되면 하나로 합칩니다.

### 2-2. 시그널 강도 평가
- 복수 카테고리에서 동시 포착 = 🔴 **강함**
- 단일 카테고리 + 주목할 만한 규모/내용 = 🟡 **보통**
- 단일 카테고리 + 소규모 = 참고용

### 2-3. Deal DB 크로스체크
기존 Deal DB에 등록된 기업인지 확인.

### 2-4. 투자사 적합도 태깅
- MEMORY.md 선호 프로파일 + CLAUDE.md 섹터·스테이지 기준 적용
- MEMORY.md **Drop 패턴** 참조 → 매칭 시 적합도 가중치 하향 반영
- 적합도: ★★★(높음) / ★★☆(중간) / ★☆☆(낮음)

---

## 3단계: 위클리 브리핑 작성

아래 포맷으로 전체 브리핑을 작성합니다:
> **톤**: 담백한 리서치 노트체. "~합니다/됩니다" 대신 "~임/~함/~중" 명사형 종결 사용. 과잉 수식·헤징 없이 팩트+수치 중심.

```
## Signal Radar Weekly — {날짜} (W{주차})

### Top Signals
| # | 시그널 | 기업/섹터 | 카테고리 | 강도 | Deal DB |
|---|--------|----------|---------|------|---------|
| 1 | {시그널 요약} | {기업명} | 펀딩/인재/소셜/미디어/규제 | 🔴/🟡 | 신규/기존 Status |

### 1. 펀딩 시그널
| 기업명 | 라운드 | 금액 | 투자자 | 적합도 |
|--------|--------|------|--------|--------|
| ... | ... | ... | ... | ★★★/★★☆/★☆☆ |
- 펀딩 트렌드 요약: {이번 주 투자 집중 섹터, 특이사항}

### 2. 인재 흐름
| 인물 | 이전 소속 | 이동 | 시그널 |
|------|----------|------|--------|
| ... | ... | ... | 창업/이직/채용 |
- 인재 흐름 요약: {주요 이동 패턴}

### 3. 소셜 시그널
- GitHub Trending: {한국 관련 주목 프로젝트}
- 앱스토어: {순위 급상승 앱}
- 채용: {대규모 채용 공고 기업}

### 4. 미디어 시그널
| 기업명 | 매체 | 핵심 내용 | 날짜 |
|--------|------|----------|------|
| ... | ... | ... | ... |

### 5. 규제/정책
- {규제 변화 1 + 영향받는 섹터}
- {규제 변화 2 + 영향받는 섹터}

### 6. 주간 인사이트
- **가장 뜨거운 섹터**: {이번 주 시그널이 집중된 섹터}
- **숨은 시그널**: {아직 주목받지 않지만 의미있는 움직임}
  - MEMORY.md Watchlist 기업과 교차 확인: 워치리스트 기업이 이번 주 시그널에 등장하면 별도 표시
- **모니터링 제안**: {다음 주 주시할 기업/섹터}

### 7. 후속 액션 제안
- [ ] {미팅 요청 추천 기업}
- [ ] {추가 리서치 필요 섹터}
- [ ] {팀 내 공유/논의 필요 사항}
```

---

## 4단계: Notion 저장
- Thesis DB에 새 페이지 생성
  - `Name`: `Signal Radar — {날짜} (W{주차})`
  - `Tags`: `signal-radar`
  - `Created Date`: 오늘 날짜

## 5단계: 터미널 전문 출력
Notion 저장 내용과 동일한 전문 마크다운 출력 + Notion 페이지 링크.

- 반론·리스크 반드시 포함
